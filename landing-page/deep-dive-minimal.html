The Problem

AI 模型的底层缺陷

即使是 GPT-4 或 Claude 3.5 Opus，也无法逃脱物理定律的束缚。

注意力稀释 (Attention Dilution)

就像人类的工作记忆一样，AI 的注意力机制是有限的。当你试图在一个 Prompt 中塞入 50 条规则时，模型在多轮对话后会不可避免地发生"灾难性遗忘"。

研究数据表明，当 System Prompt 超过 2000 token 时，末尾指令的遵循率会下降 30% 以上。这就是为什么"全能 Prompt"通常效果不佳。

上下文窗口硬限制

即使是 200K 的窗口也经不起真实项目的消耗。当代码库被读取，窗口迅速填满。超出部分会被直接丢弃，导致 AI "失忆"。

  • 任务丢失：忘记要做什么
  • 规则丢失：忘记怎么做（代码规范失效）
  • 重复工作：忘记已经修改过哪些文件

prompt_append：精准的知识注入

为了避免注意力稀释，我们不把所有规则一次性给 AI。而是根据角色动态追加 (Append) 最相关的知识。

角色：视觉代理
+ 设计系统

注入 Tailwind 颜色变量、Glassmorphism CSS 类名、间距规范、字体配置。

角色：后端代理
+ 数据库结构

注入 SQL 表结构、ORM 模型定义、API 接口规范、鉴权中间件逻辑。

角色：测试代理
+ 测试框架

注入 Pytest 配置、Playwright 选择器策略、Mock 数据生成规则。

防失忆机制

任务续接钩子

每次 AI 回复后，系统会自动检测 `todos` 列表的状态。如果发现有 `pending` 或 `in_progress` 的任务，系统会强制注入一条新的 User 消息：

"Checking todos... Found 2 pending tasks. Please continue to the next step immediately."

死循环执行器

一个死循环脚本，直到目标达成才停止。它解决了 AI "做完一步就停下来等表扬" 的惰性问题。
